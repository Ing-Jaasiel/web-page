Cable de red conectado pero sin Internet – Soluciones paso a paso

Uno de los problemas más comunes en redes domésticas y empresariales es que el cable de red está conectado, el equipo detecta la conexión, pero no hay acceso a Internet. Este problema puede ocurrir en computadoras, DVR, servidores, cámaras IP o cualquier dispositivo conectado por cable.

En esta guía aprenderás a diagnosticar la causa paso a paso como un técnico profesional, desde lo más básico hasta configuraciones avanzadas.

1. Verificar si el cable está bien ponchado

El error más frecuente en instalaciones es un mal conector RJ45. Aunque el equipo detecte el cable, algunos hilos pueden no hacer buen contacto.

Revisa los siguientes puntos:

  • Que los colores sigan el estándar T568A o T568B.
  • Que ambos extremos estén configurados con el mismo estándar.
  • Que los cables lleguen completamente hasta el fondo del conector.
  • Que no existan pines doblados o flojos.
  • Que el conector esté bien crimpado.

Consejo profesional: utiliza un probador de cable de red para verificar continuidad. Esta herramienta permite detectar errores rápidamente.

2. Probar con otro cable

Aunque el cable parezca en buen estado, puede tener daños internos por dobleces, humedad o mala calidad.

Realiza esta prueba:

  • Cambia el cable por uno nuevo o probado.
  • Si el problema desaparece, el cable original está defectuoso.

Este problema es muy común en instalaciones antiguas o con cables económicos.

3. Verificar la tarjeta de red del equipo

En computadoras o servidores:

  • Revisa que el adaptador de red esté habilitado.
  • Actualiza los controladores.
  • Desinstala y reinstala el driver si es necesario.
  • Prueba otro puerto de red.

En DVR, cámaras o switches:

  • Verifica que el LED del puerto esté encendido.
  • Revisa que el puerto no esté dañado.

4. Configuración IP incorrecta

Si la red no está bien configurada, el equipo no tendrá acceso a Internet aunque esté conectado.

Revisa:

  • Dirección IP.
  • Máscara de red.
  • Gateway.
  • DNS.

Puedes probar:

  • Obtener IP automática (DHCP).
  • Configurar manualmente según tu red.

Ejemplo:

  • IP: 192.168.1.50
  • Máscara: 255.255.255.0
  • Gateway: 192.168.1.1
  • DNS: 8.8.8.8

5. DHCP desactivado en el router

Si el servidor DHCP está desactivado, los dispositivos no obtendrán dirección IP automáticamente.

Solución:

  • Accede al router.
  • Activa el servicio DHCP.
  • Reinicia los dispositivos.

Este problema es frecuente en redes configuradas manualmente o empresariales.

6. Problemas en el switch de red

El switch puede estar defectuoso, saturado o mal configurado.

Pruebas recomendadas:

  • Conectar el equipo directamente al router.
  • Cambiar de puerto en el switch.
  • Reiniciar el switch.
  • Probar con otro switch.

7. VLAN mal configurada

En redes empresariales, el puerto del switch puede pertenecer a una VLAN que no tenga acceso a Internet.

Esto provoca:

  • Conexión local sin salida.
  • Acceso limitado a recursos.

Solución:

  • Revisar la configuración del switch.
  • Asignar la VLAN correcta.

8. Gateway incorrecto

Si el gateway está mal configurado, el equipo no podrá salir a Internet.

Ejemplo correcto:

  • IP: 192.168.1.20
  • Gateway: 192.168.1.1

9. Firewall o antivirus bloqueando

Algunos antivirus o firewalls pueden bloquear la conexión.

Prueba:

  • Desactivar temporalmente el firewall.
  • Verificar reglas de red.

10. Reinicio completo de la red

Un reinicio puede solucionar fallas temporales:

  1. Apaga el router.
  2. Apaga el switch.
  3. Apaga el equipo.
  4. Enciende primero el router.
  5. Después el switch.
  6. Por último el equipo.

Consejos de experto

Basado en experiencia real en cableado estructurado:

  • Siempre prueba con otro cable.
  • Usa herramientas profesionales.
  • Documenta direcciones IP.
  • Etiqueta el cableado.
  • Evita mezclas de estándares.
  • Realiza mantenimiento preventivo.

¿Necesitas ayuda profesional?

Si el problema continúa, puede ser necesario realizar un diagnóstico profesional. Puedes contactarnos para soporte técnico, configuración o instalación de cámaras de seguridad.
